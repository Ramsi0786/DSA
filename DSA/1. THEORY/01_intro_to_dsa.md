# 01 — What is Data Structure & Algorithms?

---

## Data Structure

A **Data Structure** is a way of **organizing and storing data** in a computer so that we can use it easily and efficiently.

**Simple words:** Just like we arrange books on a shelf to find them easily, we arrange data in a computer in a specific way using data structures.

**Examples:**
- Array → like a row of boxes, each holding one value
- Stack → like a stack of plates, last in first out
- Queue → like a line at a counter, first in first out
- Tree → like a family tree, data in hierarchy
- Graph → like a map of cities connected by roads

---

## Algorithm

An **Algorithm** is a **step-by-step set of instructions** written to solve a problem.

**Simple words:** An algorithm is like a recipe. Just like a recipe tells you exactly what to do step by step to cook food, an algorithm tells the computer exactly what to do step by step to solve a problem.

**Example:**
To find the biggest number in a list:
1. Take the first number as biggest
2. Compare it with the next number
3. If next number is bigger, update biggest
4. Repeat until end of list
5. Print the biggest number

---

## Why We Need Data Structures and Algorithms Together

- **Data Structure** = how we **store** data
- **Algorithm** = how we **process** that data

**Simple words:** Data structure is the shelf, algorithm is the method of finding a book on that shelf. Both are needed together to write good programs.

---

## Time Complexity and Space Complexity

**Time Complexity** means **how much time** an algorithm takes to run.

**Space Complexity** means **how much memory** an algorithm uses.

**Simple words:** If two methods solve the same problem, the one that takes less time and less memory is the better one. We measure this using **Big O Notation**.

---

## Big O Notation

**Big O Notation** is a way to **express how efficient** an algorithm is.

**Simple words:** It tells us how the time or memory of an algorithm grows as the input size grows.

**Common examples:**
- O(1) → constant time, very fast
- O(n) → time grows with input size
- O(log n) → time grows slowly, efficient
- O(n²) → time grows very fast, slow for large input

## Viva Q&A

**Q1. What is a Data Structure?**
A Data Structure is a way of organizing and storing data in a computer so that it can be accessed and used efficiently. Examples are Array, Stack, Queue, Tree, and Graph.

**Q2. What is an Algorithm?**
An Algorithm is a step-by-step set of instructions to solve a problem. It is like a recipe that tells the computer exactly what to do to get the result.

**Q3. What is the difference between Data Structure and Algorithm?**
A Data Structure is used to store and organize data, while an Algorithm is used to process and operate on that data. Data structure is the shelf, algorithm is the method of finding something on that shelf.

**Q4. What is Time Complexity?**
Time Complexity is a measure of how much time an algorithm takes to run as the size of input increases.

**Q5. What is Space Complexity?**
Space Complexity is a measure of how much memory an algorithm uses as the size of input increases.

**Q6. What is Big O Notation?**
Big O Notation is a standard way to express the efficiency of an algorithm. It describes how the time or space of an algorithm grows with the input size.

**Q7. Why do we study Data Structures and Algorithms?**
We study DSA to write efficient programs, to solve complex problems, to understand how software works internally, and to crack coding interviews at top companies.

**Q8. Give a real-world example of a Data Structure.**
A Queue is like a line at a billing counter. The person who comes first gets served first. This is called First In First Out (FIFO).

**Q9. Give a real-world example of an Algorithm.**
Finding the shortest route on Google Maps is an example of an algorithm. It processes the map data step by step and gives the best path.

**Q10. What happens if we use a bad Data Structure or Algorithm?**
If we use a bad data structure or algorithm, the program will be slow, use too much memory, and may even crash for large inputs.
